Collision Detection Optimisation Dissertation 2025
Master's level extended scientific research paper

For the final part of my master's degree, I undertook a 3-week-long research project investigating the effect of several collision detection optimisations. This included analysing common techniques mathematical impact, applying them programmatically, experimenting with my own tweaks, measuring the real impact and evaluating the results. This project was programmed using a university-provided bespoke basic game engine library using C++. No physics libraries were used - All physics calculations were programmed by myself. This project was marked as a first class paper.

Optimisations investigated:
  • Passive-collision culling
  • Static-collision culling
  • Bounding AABB volumes
  • Bounding sphere volumes
  • Quadtrees and octrees
  • k-d trees
Dungeon Crawler 2025
Master's level group project

For 9 weeks, I worked alongside 7 other master's degree students to produce a 3D, cross-platform, multiplayer game. We used a university-provided library to create our own engine, complete with a component system, an event system, scene hierarchy, modular saving and a world editor. Only we focused so hard on our engine that our actual game didn't turn out to be very fun.

Most of my contributions to this project were physics-based. When we started, we only had some very basic AABB collision detection and, quite literally, no optimisation. However, I also ended up designing the procedural generation strategy, which needed to use collision detection instead of a grid-based layout. To achieve this, I had to program quaternion mathematics to check for collisions with different orientations of room prefabs in the dungeon.

Engine features I programmed:
  • Event and scene systems
  • OBB collision detection
  • Collision spatial acceleration
  • Procedural generation algorithm
  • Quaternion mathematics
  • Transform calculations handling inheritance
One Cute Tabby, Not Too Shabby 2024
Advanced game technologies

For the final taught content module of my master's degree, I was taught about how game engines implement core game technologies, such as physics, AI and networking. My coursework was to take inspiration from the game "Little Kitty, Big City" (hence, the pun name) and design a basic game that included such technologies.

Features included:
  • CMake usage
  • Collision detection and resolution
  • Application of forces
  • Function pointer AI behaviour
  • State machines
  • Screen management using pushdown automata
